# Sprint File Naming Convention

Use one file per sprint with this format:

`sprint-XX-YYYY-MM-DD-short-topic.md`

Examples:
- `sprint-01-2026-03-09-website-rebuild.md`
- `sprint-02-2026-03-10-backend-proxy.md`

Rules:
- `XX` is a two-digit sprint number (`01`, `02`, `03`, ...).
- `YYYY-MM-DD` is the sprint start date.
- `short-topic` is a concise lowercase slug with hyphens.
- Keep sprint status and deliverables inside each sprint file.
