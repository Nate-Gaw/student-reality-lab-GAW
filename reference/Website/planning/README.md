# Website Planning Archive

This folder documents major past changes in sprint format.

## Sprint Files

1. `sprint-01-foundation.md`
   - Initial website structure, visual direction, and baseline interactivity.
2. `sprint-02-data-and-model.md`
   - CSV ingestion, data normalization, and ROI/payback projection logic.
3. `sprint-03-ui-iteration.md`
   - Chart order/section changes, card simplification, and data source loading updates.
4. `sprint-04-release-and-presentation.md`
   - Final polish, presentation alignment, line-ending/git consistency, and release prep.
5. `sprint-05-cli-toolkit-and-api-changes.md`
   - Cross-project changes that impacted delivery: CLI toolkit stabilization, dependency conflicts, and Gemini to OpenAI transition.

## Common Sprint Template

Each sprint includes:
- Objective
- Reason
- Scope and major changes
- Issues encountered
- Tests and validation
- Risks and follow-ups
- Files impacted

## Accuracy Notes

- This archive includes both Website-only sprints and one cross-project sprint for related CLI/tooling issues discussed during the same delivery cycle.
- The goal is traceability of major implementation changes and blockers, not a complete commit-by-commit changelog.
